# unitree-g1-experiments

Basic robot actions with the Unitree G1 Humanoid Robot (edu version).

This project reads the robot camera stream, runs person detection, and triggers a wave response and audio prompt when a person is detected.

## Requirements

- Linux
- Python 3.10 or newer
- A working Unitree SDK environment on the robot or device connected to the robot network
- GStreamer support for OpenCV video capture

## Install

Create and activate a virtual environment, then install the Python dependencies:

```bash
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install --upgrade pip
python3 -m pip install -r requirements.txt
```

If `unitree-sdk2py` is not available from PyPI in your environment, install it the same way you normally install the Unitree SDK Python package on your device, then rerun the `pip install -r requirements.txt` command for the remaining dependencies.

## Run

Use the robot network interface name that reaches the Unitree G1 stream:

```bash
PYTHONPATH=/usr/lib/python3/dist-packages python3 main.py enp130s0
```

You can also pass the detector and scenario explicitly:

```bash
PYTHONPATH=/usr/lib/python3/dist-packages python3 main.py enp130s0 --detector yolo_face --scenario welcome_wave
```

## What it does

- Connects to the robot SDK
- Opens the multicast camera stream
- Runs YOLO-based person detection
- Wakes the robot arm with `WaveHand()` when a person is confirmed
- Speaks a short audio prompt

## Project layout

- `main.py` is the entry point.
- `experiment/runner.py` contains the main loop.
- `experiment/clients.py` initializes the Unitree SDK clients.
- `experiment/detectors/` contains detection implementations.
- `experiment/scenarios/` contains activity sequences.

## Notes

- `debug.jpg` is generated by the loop and is ignored by Git.
- If the detector fires too easily, increase `detection_confidence` in `experiment/config.py`.
- If the robot reacts too often, increase `alert_interval_seconds` or `confirmation_frames`.
